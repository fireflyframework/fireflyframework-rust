# `firefly-testkit`

> **Tier:** Platform · **Status:** Full · **Java original:** `spring-boot-starter-test` · **Go module:** `testkit`

## Overview

`firefly-testkit` is the framework's **shared testing toolkit** — the helpers
every `#[cfg(test)]` module in every Firefly service ends up needing:

| Group        | Helper                                                                  |
|--------------|-------------------------------------------------------------------------|
| HMAC signers | `sign_hmac` / `sign_stripe` / `sign_github` / `sign_twilio`             |
| Event spy    | `SpyBroker::record(...)`, `.find_by_type(...)`, `.reset()`, `.len()`    |
| JSON         | `must_encode(&v)` / `must_decode(&data)`                                |

Every signer matches the wire shape of its corresponding `webhooks`
validator — drop them into a test handler and a real Stripe / GitHub /
Twilio webhook will validate identically. Signature encodings are
byte-for-byte compatible with the Go port (verified against fixtures
generated by the Go implementation).

## Public surface

```rust
// HMAC signers
pub fn sign_hmac(secret: &[u8], body: &[u8]) -> String;                 // "sha256=<hex>"
pub fn sign_stripe(secret: &[u8], body: &[u8], unix_ts: i64) -> String; // "t=<unix>,v1=<hex>"
pub fn sign_github(secret: &[u8], body: &[u8]) -> String;               // alias of sign_hmac
pub fn sign_twilio(auth_token: &[u8], post_url: &str, form: &[(&str, &str)]) -> String; // base64

// Event-recording broker
pub struct RecordedEvent { pub topic: String, pub event_type: String, pub payload: Vec<u8> }

pub struct SpyBroker { /* mutex-guarded items */ }
impl SpyBroker {
    pub fn new() -> Self;
    pub fn record(&self, topic: &str, event_type: &str, payload: &[u8]);
    pub fn find_by_type(&self, event_type: &str) -> Vec<RecordedEvent>;
    pub fn items(&self) -> Vec<RecordedEvent>;
    pub fn reset(&self);
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
}

// JSON helpers (panic on failure — panics fail the enclosing test,
// matching Go's t.Fatalf semantics)
pub fn must_encode<T: Serialize + ?Sized>(v: &T) -> Vec<u8>;
pub fn must_decode<T: DeserializeOwned>(data: &[u8]) -> T;
```

Design notes versus the Go module:

- `sign_stripe` takes the Unix-seconds timestamp directly (`i64`) instead
  of a `time.Time`; the signed payload `"<unix>.<body>"` and the header
  shape are unchanged.
- `sign_twilio` takes the form as ordered `(key, value)` pairs; keys are
  signed in sorted order and only the first value of a repeated key
  participates, mirroring Go's `url.Values.Get`. The HMAC-SHA1 it needs is
  hand-rolled (the workspace crypto catalog is SHA-2-only) and verified
  against RFC 2202 / RFC 3174 vectors plus a Go-generated fixture.
- `SpyBroker` drops Go's `context.Context` parameter (no Rust analog is
  needed) and guards its items with a `Mutex`, so a shared `Arc<SpyBroker>`
  can record from concurrent tasks.

## Quick start

Verify a webhook receiver (axum router, in-process via `tower::oneshot`):

```rust
use axum::{body::Body, http::{Request, StatusCode}};
use firefly_testkit::sign_stripe;
use tower::ServiceExt;

#[tokio::test]
async fn stripe_webhook_accepts_signed_body() {
    let secret = b"whsec_test";
    let body = br#"{"type":"charge.succeeded"}"#;

    let req = Request::post("/api/webhooks/stripe")
        .header("Stripe-Signature", sign_stripe(secret, body, 1_700_000_000))
        .body(Body::from(&body[..]))
        .unwrap();

    let res = app().oneshot(req).await.unwrap();
    assert_eq!(res.status(), StatusCode::ACCEPTED);
}
```

Assert which events a handler emitted:

```rust
use firefly_testkit::{must_encode, SpyBroker};

#[test]
fn place_order_emits() {
    let spy = SpyBroker::new();
    let place_order = |order: &Order| {
        let body = must_encode(order);
        spy.record("orders", "OrderPlaced", &body);
    };
    place_order(&sample);
    assert_eq!(spy.find_by_type("OrderPlaced").len(), 1);
}
```

## Testing

```bash
cargo test -p firefly-testkit
```

Covers each signer against known-good HMAC fixtures generated by the Go
implementation, RFC 2202/3174 vectors for the hand-rolled HMAC-SHA1,
SpyBroker record/filter/reset plus thread-safety, and the JSON helpers.
