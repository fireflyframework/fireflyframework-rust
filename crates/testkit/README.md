# `firefly-testkit`

> **Tier:** Platform · **Status:** Full · **Java original:** `spring-boot-starter-test` · **Go module:** `testkit`

## Overview

`firefly-testkit` is the framework's **shared testing toolkit** — the helpers
every `#[cfg(test)]` module in every Firefly service ends up needing:

| Group            | Helper                                                                  | Feature     |
|------------------|-------------------------------------------------------------------------|-------------|
| HMAC signers     | `sign_hmac` / `sign_stripe` / `sign_github` / `sign_twilio`             | *(default)* |
| Event spy        | `SpyBroker::record(...)`, `.find_by_type(...)`, `.reset()`, `.len()`    | *(default)* |
| Event assertions | `assert_event_published` / `assert_event_published_with` / `assert_no_events_published` | *(default)* |
| JSON             | `must_encode(&v)` / `must_decode(&data)`                                | *(default)* |
| HTTP test client | `TestClient` / `TestResponse` — in-process over an axum `Router`        | `web`       |
| DI test slices   | `Slice` / `BuiltSlice` — subset + `Arc` overrides, eager resolve        | `container` |
| Testcontainers   | `ServiceContainer` / `config_for` / `config_for_all` / `docker_available` | `testcontainers` |

The default surface carries no heavy dependencies, so a service that only needs
the signers gets a lean build. The richer migration-ergonomics helpers — the
Rust analogs of pyfly's `PyFlyTestClient`, `slice_context`/`mock_bean`, and
`testcontainers` — are opt-in behind the `web`, `container`, and
`testcontainers` features.

Every signer matches the wire shape of its corresponding `webhooks`
validator — drop them into a test handler and a real Stripe / GitHub /
Twilio webhook will validate identically. Signature encodings are
byte-for-byte compatible with the Go port (verified against fixtures
generated by the Go implementation).

## Cargo features

```toml
[dev-dependencies]
firefly-testkit = { version = "26.6", features = ["web", "container", "testcontainers"] }
```

- **`web`** — the in-process `TestClient` over an axum `Router` (pulls in
  `axum` / `tower` / `http` / `http-body-util` + a Tokio runtime).
- **`container`** — the DI `Slice` builder over `firefly-container`.
- **`testcontainers`** — the integration-test fixtures (`ServiceContainer` →
  `config_for` → `firefly.*` config keys, plus `docker_available`). Pulls in
  `firefly-config` so the overrides convert to a real `StaticSource`. It takes
  the connection URL / `(host, port)` handle any container library already
  hands you, so it adds **no** container-runtime dependency.

All three are off by default; the signer / `SpyBroker` / event-assertion / JSON
surface is always compiled with no extra dependencies.

## Public surface

```rust
// HMAC signers
pub fn sign_hmac(secret: &[u8], body: &[u8]) -> String;                 // "sha256=<hex>"
pub fn sign_stripe(secret: &[u8], body: &[u8], unix_ts: i64) -> String; // "t=<unix>,v1=<hex>"
pub fn sign_github(secret: &[u8], body: &[u8]) -> String;               // alias of sign_hmac
pub fn sign_twilio(auth_token: &[u8], post_url: &str, form: &[(&str, &str)]) -> String; // base64

// Event-recording broker
pub struct RecordedEvent { pub topic: String, pub event_type: String, pub payload: Vec<u8> }

pub struct SpyBroker { /* mutex-guarded items */ }
impl SpyBroker {
    pub fn new() -> Self;
    pub fn record(&self, topic: &str, event_type: &str, payload: &[u8]);
    pub fn find_by_type(&self, event_type: &str) -> Vec<RecordedEvent>;
    pub fn items(&self) -> Vec<RecordedEvent>;
    pub fn reset(&self);
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
}

// Event-emission assertions on a SpyBroker (panic on failure)
pub fn assert_event_published(spy: &SpyBroker, event_type: &str) -> RecordedEvent;
pub fn assert_event_published_with(spy: &SpyBroker, event_type: &str, payload_contains: &Value) -> RecordedEvent;
pub fn assert_no_events_published(spy: &SpyBroker);

// JSON helpers (panic on failure — panics fail the enclosing test,
// matching Go's t.Fatalf semantics)
pub fn must_encode<T: Serialize + ?Sized>(v: &T) -> Vec<u8>;
pub fn must_decode<T: DeserializeOwned>(data: &[u8]) -> T;

// `web` feature — in-process HTTP test client over an axum Router
pub struct TestClient { /* clone-cheap Router wrapper */ }
impl TestClient {
    pub fn new(router: axum::Router) -> Self;
    // async (call from a #[tokio::test])
    pub async fn get(&self, uri: &str) -> TestResponse;
    pub async fn delete(&self, uri: &str) -> TestResponse;
    pub async fn post<T: Serialize + ?Sized>(&self, uri: &str, body: &T) -> TestResponse;
    pub async fn put<T: Serialize + ?Sized>(&self, uri: &str, body: &T) -> TestResponse;
    pub async fn patch<T: Serialize + ?Sized>(&self, uri: &str, body: &T) -> TestResponse;
    pub async fn post_empty(&self, uri: &str) -> TestResponse;
    pub async fn request(&self, method: Method, uri: &str, content_type: Option<&str>, body: Body) -> TestResponse;
    // blocking (call from a plain #[test] — reads like pyfly's sync client)
    pub fn get_blocking(&self, uri: &str) -> TestResponse;       // + delete/post/put/patch/post_empty_blocking
}

pub struct TestResponse { /* buffered status + headers + body */ }
impl TestResponse {
    pub fn status(&self) -> u16;
    pub fn header(&self, name: &str) -> Option<&str>;            // case-insensitive
    pub fn text(&self) -> String;
    pub fn body_bytes(&self) -> &[u8];
    pub fn json<T: DeserializeOwned>(&self) -> T;
    // chainable assertions (each returns &Self)
    pub fn assert_status(&self, expected: u16) -> &Self;
    pub fn assert_success(&self) -> &Self;
    pub fn assert_header(&self, name: &str, value: &str) -> &Self;
    pub fn assert_header_present(&self, name: &str) -> &Self;
    pub fn assert_body_contains(&self, needle: &str) -> &Self;
    pub fn assert_json_eq(&self, expected: &Value) -> &Self;
    // JSONPath single-result subset ($, $.a.b, $['a'], $[0], $.items[1].id)
    pub fn json_path(&self, path: &str) -> Option<Value>;
    pub fn assert_json_path(&self, path: &str, expected: impl Into<Value>) -> &Self;
    pub fn assert_json_path_exists(&self, path: &str) -> &Self;
    pub fn assert_json_path_absent(&self, path: &str) -> &Self;
}

// `testcontainers` feature — map a started backing service to framework config
pub enum ServiceContainer { Postgres { .. }, MySql { .. }, Redis { .. }, MongoDb { .. }, Kafka { .. }, RabbitMq { .. } }
impl ServiceContainer {
    pub fn postgres_at(host, port, user, password, database) -> Self;   // + mysql/redis/mongodb/kafka/rabbitmq_at
    pub fn from_url(url: &str) -> Result<Self, String>;                 // infer from a connection URL scheme
}

pub fn config_for(container: &ServiceContainer) -> ConfigOverrides;     // the @ServiceConnection mapper
pub fn config_for_all<'a>(it: impl IntoIterator<Item = &'a ServiceContainer>) -> ConfigOverrides;
pub fn docker_available() -> bool;                                      // the requires_docker skip guard

pub struct ConfigOverrides { /* flat firefly.* dotted keys */ }
impl ConfigOverrides {
    pub fn get(&self, key: &str) -> Option<&str>;
    pub fn as_map(&self) -> &BTreeMap<String, String>;
    pub fn extend(&mut self, other: &ConfigOverrides);
    pub fn into_source(self) -> firefly_config::StaticSource;           // drop into firefly-config Layered
}

// `container` feature — DI test slice (subset of beans + Arc overrides)
pub struct Slice { /* Container builder */ }
impl Slice {
    pub fn new() -> Self;
    pub fn register<T, F>(self, scope: Scope, factory: F) -> Self;          // a collaborator
    pub fn register_named<T, F>(self, scope: Scope, name: &str, factory: F) -> Self;
    pub fn register_scoped<T, F>(self, scope: impl Into<ScopeSpec>, name: &str, factory: F) -> Self;
    pub fn instance<T>(self, instance: T) -> Self;                          // override / mock-bean analog
    pub fn instance_named<T>(self, instance: T, name: &str) -> Self;
    pub fn bind<I, T>(self, coerce: impl Fn(Arc<T>) -> Arc<I>) -> Self;     // port -> fake
    pub fn eager<T>(self) -> Self;                                          // fail-fast gate
    pub fn container(&self) -> &Container;
    pub fn try_build(self) -> Result<BuiltSlice, ContainerError>;
    pub fn build(self) -> BuiltSlice;                                       // panics on eager-resolve failure
}

pub struct BuiltSlice { /* Arc<Container> */ }
impl BuiltSlice {
    pub fn get<T>(&self) -> Arc<T>;
    pub fn get_named<T>(&self, name: &str) -> Arc<T>;
    pub fn container(&self) -> Arc<Container>;
}
```

Design notes versus the Go module:

- `sign_stripe` takes the Unix-seconds timestamp directly (`i64`) instead
  of a `time.Time`; the signed payload `"<unix>.<body>"` and the header
  shape are unchanged.
- `sign_twilio` takes the form as ordered `(key, value)` pairs; keys are
  signed in sorted order and only the first value of a repeated key
  participates, mirroring Go's `url.Values.Get`. The HMAC-SHA1 it needs is
  hand-rolled (the workspace crypto catalog is SHA-2-only) and verified
  against RFC 2202 / RFC 3174 vectors plus a Go-generated fixture.
- `SpyBroker` drops Go's `context.Context` parameter (no Rust analog is
  needed) and guards its items with a `Mutex`, so a shared `Arc<SpyBroker>`
  can record from concurrent tasks.

Design notes versus the pyfly testing package:

- `assert_event_published(*)` take a `&SpyBroker` rather than pyfly's
  `list[EventEnvelope]`; `assert_event_published_with` performs the same
  *subset* JSON-payload check as pyfly's `payload_contains=` (extra keys in the
  event are ignored). All three panic on mismatch — the Rust analog of pyfly's
  `AssertionError` — with the same "published types"/"expected key" diagnostics.
- `TestClient` is the Rust `PyFlyTestClient`: it drives an axum `Router`
  *in process* via `tower::ServiceExt::oneshot` (no socket bound, no server
  task). Each method has an `async` form (for `#[tokio::test]`) and a
  `*_blocking` form (for a plain `#[test]`) that runs the request on an internal
  shared current-thread runtime so it reads exactly like pyfly's synchronous
  `client.get(...)`. `TestResponse`'s assertions are chainable like pyfly's.
- `Slice` is the Rust `slice_context` + `mock_bean`. Rust has no package
  scanning or reflective autowiring, so a slice is an explicit builder: list the
  bean factories and pre-built overrides, mark beans `eager`, and `build()`
  resolves them so a missing collaborator fails at *build* time (the
  `ContainerError` carries the container's fuzzy suggestions). `instance(fake)` +
  `bind::<dyn Port, Fake>(|a| a)` overrides a port with a fake; the container
  owns the resulting `Arc`, so `slice.get::<Fake>()` returns the very same handle
  to configure (via interior mutability) and assert against — the `mock_bean`
  use-case.
- `TestResponse::assert_json_path` is the Rust `assert_json_path`: a
  self-contained JSONPath selector over the **single-result subset** real tests
  use (`$`, `$.a.b`, `$['a']`, `$[0]`, `$.items[1].id`) — no `jsonpath-ng`
  dependency. `assert_json_path(path, value)`, `assert_json_path_exists` and
  `assert_json_path_absent` cover pyfly's `value=` / `exists=True|False` forms;
  unsupported grammar (wildcards, recursive descent, filters) panics loudly.
- `ServiceContainer` + `config_for` are the Rust `testcontainers` fixtures —
  the `@ServiceConnection` / `pyfly_config_for` / `pyfly_config` equivalents.
  Rather than wrap one container runtime, they take the connection details any
  library hands you (a URL via `from_url`, or `(host, port)` via `*_at`) and map
  them to the canonical `firefly.*` config keys: Postgres/MySQL →
  `firefly.data.url` + `firefly.datasource.url`, Redis →
  `firefly.cache.redis.url` + `firefly.session.redis.url`, MongoDB →
  `firefly.data.url` + `firefly.data.document.uri`, Kafka →
  `firefly.eda.kafka.bootstrap-servers`, RabbitMQ → `firefly.eda.rabbitmq.url` +
  `firefly.messaging.rabbitmq.url`. `config_for_all` merges several (pyfly's
  `pyfly_config(*containers)`); `into_source()` yields a `firefly-config`
  `StaticSource` to drop into the layered config; `docker_available()` is the
  `requires_docker` skip guard.

## Quick start

Verify a webhook receiver (axum router, in-process via `tower::oneshot`):

```rust
use axum::{body::Body, http::{Request, StatusCode}};
use firefly_testkit::sign_stripe;
use tower::ServiceExt;

#[tokio::test]
async fn stripe_webhook_accepts_signed_body() {
    let secret = b"whsec_test";
    let body = br#"{"type":"charge.succeeded"}"#;

    let req = Request::post("/api/webhooks/stripe")
        .header("Stripe-Signature", sign_stripe(secret, body, 1_700_000_000))
        .body(Body::from(&body[..]))
        .unwrap();

    let res = app().oneshot(req).await.unwrap();
    assert_eq!(res.status(), StatusCode::ACCEPTED);
}
```

Assert which events a handler emitted:

```rust
use firefly_testkit::{assert_event_published_with, assert_no_events_published, must_encode, SpyBroker};

#[test]
fn place_order_emits() {
    let spy = SpyBroker::new();
    spy.record("orders", "OrderPlaced", &must_encode(&serde_json::json!({ "id": 1, "total": 42 })));

    // subset payload match; returns the matching RecordedEvent for further checks.
    let event = assert_event_published_with(&spy, "OrderPlaced", &serde_json::json!({ "id": 1 }));
    assert_eq!(event.topic, "orders");

    spy.reset();
    assert_no_events_published(&spy);
}
```

Drive an axum `Router` in process (`web` feature):

```rust
use axum::{routing::get, Json, Router};
use firefly_testkit::TestClient;

#[tokio::test]
async fn lists_users() {
    let app: Router = build_app();
    let client = TestClient::new(app);

    client
        .get("/api/users")
        .await
        .assert_status(200)
        .assert_header("content-type", "application/json")
        .assert_json_eq(&serde_json::json!([{ "name": "Alice" }]));
}
```

Build a DI slice with a fake collaborator (`container` feature):

```rust
use firefly_testkit::Slice;
use firefly_container::Scope;

#[test]
fn service_with_fake_repo() {
    let slice = Slice::new()
        .instance(FakeRepo::default())              // the mock-bean analog
        .bind::<dyn Repo, FakeRepo>(|a| a)          // override the `dyn Repo` port
        .register::<OrderService, _>(Scope::Singleton, |c| {
            Ok(OrderService { repo: c.resolve::<dyn Repo>()? })
        })
        .eager::<OrderService>()                    // fail fast if `Repo` is missing
        .build();

    let svc = slice.get::<OrderService>();
    svc.place_order();
    // the fake retrieved from the container is the SAME instance the service wired in
    assert_eq!(slice.get::<FakeRepo>().saved(), 1);
}
```

Assert a single field deep in the response body (`web` feature):

```rust
client
    .get("/api/users")
    .await
    .assert_status(200)
    .assert_json_path("$[0].name", "Alice")     // first user's name
    .assert_json_path("$[0].roles[0]", "admin") // first role of the first user
    .assert_json_path_absent("$[2]");           // only two users
```

Wire a real backing service into framework config for an integration test
(`testcontainers` feature):

```rust
use firefly_testkit::containers::{config_for_all, docker_available, ServiceContainer};
use firefly_config::{Layered, Source, StaticSource};

#[tokio::test]
async fn boots_against_real_postgres_and_redis() {
    if !docker_available() {
        return; // skip cleanly where no container runtime is reachable
    }
    // Start containers with your library of choice, then wrap their host+port:
    let pg = ServiceContainer::postgres_at("127.0.0.1", pg_port, "app", "secret", "app");
    let redis = ServiceContainer::redis_at("127.0.0.1", redis_port);

    // The @ServiceConnection step: container details -> firefly.* config keys.
    let overrides = config_for_all([&pg, &redis]);
    let layered = Layered::new(vec![
        Box::new(app_defaults()),
        Box::new(overrides.into_source()), // container URLs win over defaults
    ]);
    let flat = layered.map().unwrap();
    assert!(flat["firefly.data.url"].starts_with("postgres://"));
    assert!(flat["firefly.cache.redis.url"].starts_with("redis://"));
    // ... boot the ApplicationContext from `flat` and run the test ...
}
```

## Testing

```bash
cargo test -p firefly-testkit                 # default surface
cargo test -p firefly-testkit --all-features  # + TestClient + Slice + testcontainers
```

Covers each signer against known-good HMAC fixtures generated by the Go
implementation, RFC 2202/3174 vectors for the hand-rolled HMAC-SHA1,
SpyBroker record/filter/reset plus thread-safety, the JSON helpers, the event
assertions (match / subset / missing-key / mismatch / non-JSON paths), the
`TestClient` across every verb plus path params, headers, error bodies, the
blocking wrappers and clone-reuse, the `assert_json_path` selector (array index,
nested members, bracketed access, whole-document `$`, missing/absent paths,
unsupported-grammar panics), the `Slice` builder (subset resolution, shared
overrides, fail-fast eager resolve, named beans), and the `testcontainers`
fixtures (each service's config-key mapping, `from_url` scheme inference,
multi-container merge, `into_source`, and the `docker_available` decision).
