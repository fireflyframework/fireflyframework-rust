# Gap report: pyfly `web`

Verdict: **partial** · est. 1000 LOC

## What it is
pyfly.web is the framework's full Spring-WebMVC-equivalent HTTP layer (~5,900 LOC, all FULL implementations, no stubs): a Starlette (and alternate FastAPI) app factory that auto-discovers @rest_controller beans, binds handler parameters from type annotations (PathVar/QueryParam/Body/Header/Cookie/File/Valid), and runs an ordered 12-filter chain (request context, correlation/W3C trace headers, OTel span, transaction id, request logging, security headers, Micrometer-parity HTTP metrics, CSRF, JWT/OAuth2 bearer auth, URL-rule authorization, Idempotency-Key replay). It also provides global exception handling (error envelope + opt-in RFC 7807) with a pluggable exception-converter chain, JSON/XML content negotiation via a message-converter registry, a central JSON serialization layer, OpenAPI 3.1 generation with Swagger UI/ReDoc, and SSE endpoints. Much of its machinery (decorator reflection, bean auto-configuration, the dual FastAPI adapter, contextvars plumbing) is Python-runtime-specific with native axum/serde/tower analogs.

## pyfly public surface
pyfly.web: create_app (Starlette + FastAPI adapters); @request_mapping/@get|post|put|patch|delete_mapping(+status_code)/@deprecated; binding markers PathVar/QueryParam/Body/Header/Cookie/File/Valid + UploadedFile; WebFilter protocol + OncePerRequestFilter (url/exclude glob patterns) and built-in filters: RequestContextFilter, CorrelationFilter (X-Correlation-Id/X-Request-Id/X-Tenant-Id/traceparent/tracestate), TracingFilter (OTel SERVER span), TransactionIdFilter, RequestLoggingFilter, SecurityHeadersFilter, MetricsFilter (http_server_requests_seconds + _max), CsrfFilter, SecurityFilter, OAuth2ResourceServerFilter, HttpSecurityFilter, IdempotencyWebFilter (+@disable_idempotency); CORSConfig.from_config; SecurityHeadersConfig; global_exception_handler (error envelope or RFC 7807 problem+json, 25-type exception->status map) + ExceptionConverter chain; @exception_handler/@controller_advice; MessageConverter/JsonMessageConverter/XmlMessageConverter/MessageConverterRegistry/parse_accept; JSON layer CamelModel/JsonProperties/JsonSerializers/PyFlyJsonSerializer; OpenAPIGenerator (3.1, schemas from type hints) + /openapi.json /docs /redoc endpoints; SSE: @sse_mapping, format_sse_event, SseEmitter, SSERegistrar, make_sse_response; WebFilterChainMiddleware; auto-configurations (web adapter, idempotency, CSRF, HttpSecurity).

## Missing in Rust
Five middleware layers plus two helpers have no counterpart anywhere in the 52-crate workspace (verified by grep: no tower-http in the dep catalog, no cors/x-frame-options/nosniff/csrf/http_server_requests/request-logging hits outside resilience's token-bucket and kernel's 429 problem type): (1) CORS — pyfly CORSConfig + config-driven CORSMiddleware (pyfly.web.cors.*, Spring permit-default GET/HEAD/POST, preflight/max-age/credentials/expose-headers); (2) security response headers — SecurityHeadersConfig + filter (nosniff, X-Frame-Options DENY, HSTS, X-XSS-Protection 0, Referrer-Policy, optional CSP/Permissions-Policy); (3) CSRF double-submit-cookie filter (XSRF-TOKEN cookie vs X-XSRF-TOKEN header, timing-safe compare, Bearer bypass, token rotation, 403 on mismatch); (4) structured request-access-log middleware (method/path/status/duration_ms/transaction_id, error path logged); (5) HTTP server metrics middleware — Micrometer-parity http_server_requests_seconds timer with method/templated-uri/status/outcome/exception tags + time-windowed _max gauge (Rust actuator has a MetricRegistry but nothing auto-instruments HTTP). Also: (6) correlation surface is X-Correlation-Id only — pyfly additionally propagates/echoes X-Request-Id, X-Tenant-Id, X-Transaction-Id and echoes traceparent/tracestate (wire headers other ports emit); (7) Accept-header content negotiation with XML read/write (MessageConverterRegistry, q-value parsing, dict<->XML) — Rust is JSON-only; (8) ReDoc docs endpoint (firefly-openapi serves only /openapi.json + a Swagger-UI shim). Everything else is covered elsewhere: problem/RFC7807 + panic recovery (web::ProblemLayer, kernel::as_problem), idempotency replay (web::IdempotencyLayer, richer than pyfly), JWT bearer extraction with anonymous fallthrough (security::BearerLayer ≈ SecurityFilter/OAuth2ResourceServerFilter), URL-rule authorization with first-match-wins problem+json 401/403 (security::FilterChain ≈ HttpSecurityFilter), SSE wire format + Last-Event-Id (firefly-sse), OpenAPI 3.1 + ProblemDetail default responses (firefly-openapi, manual registration by design — no reflection), actuator routes (firefly-actuator), and chain assembly (starter-core Core::apply_middleware ≈ create_app). Excluded as Python-runtime machinery with native Rust analogs: mapping decorators/controller discovery/ParameterResolver (axum routing + Path/Query/Json/Multipart extractors), Valid/pydantic 422 (serde + validators crate), exception-converter chain and @exception_handler/@controller_advice (From<FireflyError> for WebError + typed errors), JsonProperties/CamelModel global JSON config (serde attrs), WebFilter/OncePerRequestFilter glob-pattern chain (tower per-route layering), auto-configuration beans (explicit wiring), and the entire FastAPI alternate adapter.

## Porting plan
Extend crates/web (firefly-web) — no new crate. Follow its existing hand-rolled tower::Layer style (the workspace deliberately avoids tower-http). Add five modules: (1) cors.rs — CorsConfig {allowed_origins, allowed_methods (default GET/HEAD/POST), allowed_headers, allow_credentials, exposed_headers, max_age: 600} + CorsLayer handling preflight OPTIONS short-circuit and response decoration; deps: http, axum (already in crate). (2) headers.rs — SecurityHeadersConfig (same 7 fields/defaults as pyfly) + SecurityHeadersLayer that pre-encodes static HeaderName/HeaderValue pairs once and appends on every response. (3) csrf.rs — CsrfLayer: double-submit XSRF-TOKEN cookie vs X-XSRF-TOKEN header, safe-method pass-through with cookie refresh, Bearer bypass, constant-time compare (compare sha2 digests with existing sha2 dep to avoid adding `subtle`), 403 problem+json via problem::problem_response. (4) request_log.rs — RequestLogLayer emitting one tracing event per request (method, path, status, duration_ms, correlation id from extensions); reuses CorrelationId extension. (5) metrics.rs — MetricsLayer recording into firefly_actuator::MetricRegistry (add firefly-actuator dep, or define a small RequestObserver trait in web and bridge in starter-core to avoid the dependency edge) with method/matched-route(axum MatchedPath)/status/outcome/exception labels + the two-window rolling max from pyfly. Extend correlation.rs to also read/generate X-Request-Id, propagate X-Tenant-Id and X-Transaction-Id, and echo traceparent/tracestate (store a CorrelationContext struct in request extensions next to CorrelationId; keep wire header names byte-identical). Wire all of it into starter-core Core::apply_middleware behind Option<CorsConfig>/Option<SecurityHeadersConfig>/bool flags on CoreConfig (config-driven enablement replaces pyfly auto-configuration). Add a ~20-line make_redoc route to firefly-openapi::Builder::router. Defer XML content negotiation: no XML dep exists in the catalog and Rust handlers return typed responses; if required later, add quick-xml to the catalog and a Negotiate<T> responder in firefly-web (~250 LOC extra, not counted). Idiom adaptations: decorator-marked filters -> explicit Layer ordering in apply_middleware; glob url_patterns -> per-route layering or a small path-predicate wrapper; @disable_idempotency -> route-scoped layer omission. Estimated ~1,000 LOC src (cors 200, headers 90, csrf 160, request_log 90, metrics 220, correlation ext 120, starter-core wiring 100, redoc 20) plus tests.
