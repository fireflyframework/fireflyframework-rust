# Gap report: pyfly `cli`

Verdict: **gap** · est. 5500 LOC

## What it is
A Click-based developer CLI (`pyfly` binary, ~3,540 src LOC + 59 Jinja2 templates / ~3,200 template lines, ~30 test files) that scaffolds new projects (7 archetypes x 15 features, interactive wizard), generates per-artifact code (service/controller/entity/repository/dto/aggregate/CQRS command+query/event/saga/scheduled/full CRUD resource), wraps Alembic migrations (`pyfly db` with 12 subcommands), and introspects apps (routes/beans/env/health/metrics/conditions via offline context boot or remote `/actuator` over `--url`, plus OpenAPI export). It also carries Python-runtime-specific machinery with no Rust analog: `pyfly run` (granian/uvicorn/hypercorn ASGI server + uvloop/winloop event-loop adapter selection, --reload file watching), `pyfly shell` (ptpython/IPython REPL with booted DI context), quality wrappers around pytest/ruff/mypy, pip/uv self-upgrade, and pyproject-extras patching — these are excluded from the gap.

## pyfly public surface
Entry point `pyfly.cli.main:cli` (Click group with banner). Commands: `new` (archetypes core|web-api|fastapi-api|web|hexagonal|library|cli; --archetype/--features/--directory/--list/--git/--no-input; interactive wizard); `generate|g` {service, controller, entity, repository, dto, aggregate, command, query, event, saga, scheduled, shell-command, resource, migration} with --force/--dry-run; `db` {init, migrate, upgrade, downgrade, current, history, heads, show, revision, stamp, merge, reset}; `run` (--host/--port/--reload/--app/--server/--workers/--profile/-D/--env/--debug/--watch); introspection {routes, beans, env, health, metrics, conditions, actuator} (--url remote or offline boot, --json); `shell` (-c); `openapi` (--format json|yaml, -o); `info`; `doctor`; `license`; `sbom` (--json); `features`/`add`/`remove`; `build` {wheel, sdist, image, info}; {test, lint, format, typecheck}; `completion` (bash|zsh|fish); `upgrade`; `plugins list` + third-party subcommands via entry-point group 'pyfly.cli_plugins'. Library helpers: templates.generate_project()/AVAILABLE_FEATURES/DEFAULT_FEATURES/ARCHETYPE_DETAILS/FEATURE_DETAILS, generate.Artifact/write_artifacts(), naming.names()/Names (pascal/snake/kebab/camel/plural/human variants), _project.detect_project()/ProjectInfo/feature_flags(), _introspect.boot_context()/ActuatorClient.

## Missing in Rust
The entire developer-CLI surface: the 52-crate workspace has no CLI crate and no [[bin]] target (only samples/orders). Concretely absent: (1) project scaffolding — no `firefly new` equivalent, no archetype/feature templates producing a runnable Cargo workspace member (Cargo.toml, main.rs, firefly.yaml, Dockerfile, tests); (2) artifact generators — no `firefly generate` for handler/route/entity/repository/dto/aggregate/CQRS command+query/event+listener/saga/scheduled-job skeletons with naming-case derivation and project detection; (3) migration tooling — firefly-migrations runs V###__name.sql files as a library at startup, but there is no CLI to create a numbered migration file, show applied/pending status, or apply/baseline from a terminal; (4) actuator introspection client — firefly-actuator serves /actuator/{health,info,metrics,env,tasks,version}, but no CLI fetches/pretty-prints them (--url --json); (5) offline OpenAPI export — firefly-openapi serves /openapi.json at runtime, but no command writes the spec to a file; (6) doctor/info/sbom-style environment diagnostics; (7) CLI plugin mechanism. Excluded as Python-runtime-specific: `run` server/event-loop adapter selection and reload, `shell` REPL, pytest/ruff/mypy wrappers (cargo test/clippy/fmt are native), pip/uv upgrade, shell completion plumbing (free with clap_complete), pyproject extras patching (cargo add --features is native).

## Porting plan
New crate `crates/cli` → package `firefly-cli` with `[[bin]] name = "firefly"` (install via cargo install). Add to workspace catalog: clap v4 (derive + clap_complete), minijinja (or embed plain `format!` templates via include_str! + rust-embed style include_dir), and optionally dialoguer for the interactive wizard; reuse existing catalog deps serde/serde_json/serde_yaml (firefly.yaml parsing), reqwest (actuator client), rusqlite (db status against firefly-migrations table), chrono (build-info, migration timestamps). Key types: `enum Archetype { Core, WebApi, Web, Hexagonal, Library, Cli }` (drop fastapi-api — single web stack in Rust), `struct Feature` catalog mapping to firefly-* crate deps + starter selection written into the generated Cargo.toml, `struct ProjectInfo { root, package, archetype, src_dir, tests_dir }` detected from Cargo.toml + firefly.yaml (port of _project.py), `struct Names { pascal, snake, kebab, camel, *_plural, human }` (port of naming.py; use heck or hand-roll ~80 LOC), `struct Artifact { kind, path, content }` + `write_artifacts(force, dry_run)` (port of generate.py core), `struct ActuatorClient { base_url }` with blocking reqwest GET /actuator/{endpoint} (port of _introspect.py remote half only — offline context boot is meaningless for a compiled binary; document `--url` as the only mode). Commands: new (template rendering with {package_name} path substitution, --git init), generate {handler, route, entity, repository, dto, aggregate, command, query, event, saga, scheduled, resource}, db {new <name> → V{next}__{name}.sql, status, apply (via firefly-migrations Database port over rusqlite/connection-string), baseline}, actuator {health, info, metrics, env, version} --url --json, openapi export (boot-free: hit a running app's /openapi.json or call firefly-openapi generator from a registration fn), doctor (rustc/cargo/git versions, workspace checks), info, completion (clap_complete). Idiom adaptations: Click decorators → clap derive subcommand enums; questionary wizard → dialoguer (or non-interactive-only first cut); Jinja2 → minijinja with the same context keys (package_name, names.*, has_* flags); entry-point plugins → external `firefly-<cmd>` binaries discovered on PATH (cargo-style subcommand convention). Estimated ~3,000 LOC of Rust + ~2,500 lines of embedded templates (6 archetypes x ~12 files + ~15 generator templates).
