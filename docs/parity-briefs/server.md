# Gap report: pyfly `server`

Verdict: **partial** · est. 550 LOC

## What it is
pyfly "server" is the application-server abstraction layer (Spring Boot WebServer analog): an ApplicationServerPort protocol with three full adapters wrapping real ASGI servers (Granian > Uvicorn > Hypercorn, chosen by conditional auto-configuration) plus an EventLoopPort with asyncio/uvloop/winloop adapters. Adapters translate a single ServerProperties config (pyfly.server.*: workers, backlog, graceful_timeout, http 1/2/auto, ssl_certfile/keyfile, keep_alive_timeout, max_concurrent_connections, max_requests_per_worker, granian.* tuning) into each server's options, and expose a frozen ServerInfo snapshot consumed by startup logging and the admin dashboard. Much of it is Python-runtime machinery with no Rust analog: multi-process workers (GIL workaround), asyncio event-loop policy selection, and the very notion of pluggable ASGI servers — in Rust there is one canonical hyper/axum/tokio stack.

## pyfly public surface
ApplicationServerPort (serve(app, config) blocking, serve_async(app, config), shutdown(), server_info property); EventLoopPort (install(), name); ServerInfo(name, version, workers, event_loop, http_protocol, host, port); GranianServerAdapter / UvicornServerAdapter / HypercornServerAdapter; AsyncioEventLoopAdapter / UvloopEventLoopAdapter / WinloopEventLoopAdapter; ServerAutoConfiguration + EventLoopAutoConfiguration (priority-ordered conditional beans); companion config: ServerProperties + GranianProperties (pyfly.server.* prefix).

## Missing in Rust
The serving lifecycle itself is covered: crates/lifecycle Application::on_server + ShutdownSignal + drain budget (default 30 s, with_drain_timeout) + SIGINT/SIGTERM trap maps serve/shutdown/graceful_timeout, and axum::serve gives HTTP/1.1 + h2c. Genuinely absent for a migrating consumer: (1) TLS termination — ssl_certfile/ssl_keyfile has no counterpart; no rustls/axum-server anywhere in the workspace, so HTTPS and ALPN-negotiated HTTP/2 are impossible; (2) a config-driven server bootstrap — no ServerProperties-equivalent struct bound from firefly-config and no helper turning it into a running server (every app hand-writes TcpListener::bind + axum::serve, e.g. samples/orders/src/main.rs); (3) connection tuning — keep_alive_timeout, backlog, max_concurrent_connections, max-requests limits are not exposed by plain axum::serve; (4) a ServerInfo runtime snapshot for /actuator/info (pyfly admin shows server name/version/host/port/protocol). Excluded as Python-only: multi-process workers, uvloop/winloop event-loop policy, and Granian/Uvicorn/Hypercorn adapter selection. HTTP/3 (Hypercorn QUIC) is also absent but is an edge feature not worth porting now.

## Porting plan
Extend crates/web (firefly-web) with a `server` module rather than a new crate, keeping crates/lifecycle dependency-light as designed. Key types: `ServerProperties` (serde Deserialize; host, port, graceful_timeout, keep_alive_timeout, backlog, max_concurrent_connections, tls: Option<TlsConfig>) bound from firefly-config under `server.*`; `TlsConfig { cert_file, key_file }`; `ServerInfo { name: "hyper", version, host, port, http_protocol, tls: bool }` registered as an actuator info contributor; and `async fn serve(router: Router, props: &ServerProperties, shutdown: ShutdownSignal) -> HookResult` (plus a `Server::bind(props)` builder) designed to drop straight into `Application::on_server`. Implementation: socket2 (new workspace dep) for backlog/SO_REUSEADDR, tower::limit::ConcurrencyLimitLayer for max connections, hyper-util auto builder for keep-alive timeouts, and a feature-gated `tls` using tokio-rustls + rustls-pemfile (new deps) wrapping the accept loop with ALPN h2/http1.1 — propagate lifecycle's drain via with_graceful_shutdown. Idiom adaptations: priority auto-configuration of adapters collapses to the single hyper stack (no port/adapter split needed); EventLoopPort maps to the tokio runtime (document `workers` -> tokio worker_threads, do not port); decorator-based conditional beans become plain Option fields on ServerProperties. pyfly tests to mirror: kwargs/option translation (ssl + tuning honored in both blocking and async paths), graceful shutdown on signal, ServerInfo population.
