# Gap report: pyfly `container`

Verdict: **gap** · est. 2100 LOC

## What it is
pyfly's Spring-style DI container (~1,630 LOC, 14 modules): type-hint-driven constructor injection with cached init plans, Autowired field injection, stereotype decorators (@component/@service/@repository/@controller/@rest_controller/@configuration/@controller_advice/@shell_component), @bean factory methods with @primary/profile, scopes (SINGLETON/TRANSIENT/REQUEST/SESSION + custom ScopeHandler SPI + RefreshScope for config reload), Qualifier/named beans, Provider[T] deferred lookup, Optional[T]/list[T]/dict[str,T] and generic-aware (Repository[User]) injection, @Value placeholder injection with coercion, package scanning with auto-binding to Protocol/ABC/ports interfaces, @order/@lazy, per-bean metrics, cycle detection, and rich diagnostics with fuzzy suggestions. It is the wiring backbone of pyfly (imported by 73 src files; ApplicationContext drives it). Python-runtime-specific machinery with NO Rust analog: importlib/pkgutil package scanning, typing.get_type_hints/inspect.signature reflection over __init__, setattr field injection, isinstance-based assignability, and MRO-walk auto-binding — these are excluded from the missing list as such; only their *outcomes* (auto-wiring, discovery) are considered.

## pyfly public surface
Container (register, register_instance, register_scope/unregister_scope, bind, resolve, resolve_by_name, resolve_all, contains, contains_type, get_registration, registered_types, reset_instance, get_bean_metrics/get_all_metrics); decorators: component, service, repository, controller, rest_controller, configuration, controller_advice, shell_component, bean(name/scope/primary/profile), primary, lazy, order (+HIGHEST/LOWEST_PRECEDENCE), refresh_scope; Autowired(qualifier, required); Qualifier; Provider[T]; Scope enum + ScopeSpec + ScopeHandler protocol; RefreshScope; Registration/BeanMetrics; scanner.scan_package/scan_module_classes; exceptions: BeanCreationException, NoSuchBeanError, NoUniqueBeanError, BeanCurrentlyInCreationError

## Missing in Rust
No crate offers any runtime service registry, bean lifecycle scopes, named-bean/qualifier resolution, deferred Provider<T>, refresh-scope (config-reload-driven instance eviction — absent from firefly-config/actuator too), per-bean metrics, or container error taxonomy (NoSuchBean/NoUniqueBean/CircularDependency). Wiring is fully explicit: starter-core's Core::new(CoreConfig) and lifecycle's hook-based Application. The only TypeId-keyed registries are message-handler-local (crates/cqrs/src/bus.rs, eda). NOTE: this absence is deliberate and precedented — DESIGN.md mandates module parity with the Go port, which also has no container module — so by the port's own success criteria nothing is owed; the gap matters only to a consumer migrating pyfly code that calls container.resolve/Provider/named beans/refresh_scope directly.

## Porting plan
Answer to the posed question: a TypeId-keyed registry mirrors the service-locator half of the pyfly surface faithfully (~60%), but cannot mirror the reflective-autowiring half without proc-macros, and even then the idiom inverts to compile-time codegen. Recommendation: keep explicit construction as primary idiom; if parity is wanted, add an OPT-IN crate firefly-container (no proc-macros in v1). Key types: Container { by_type: HashMap<TypeId, Vec<Registration>>, by_name: HashMap<String, Registration>, scopes: HashMap<String, Arc<dyn ScopeHandler>> } using RwLock interior mutability (Send+Sync, shareable as Arc<Container>); Registration { name, scope, primary, order, factory: Box<dyn Fn(&Container) -> Result<Arc<dyn Any + Send + Sync>> }, BeanMetrics, Provider<T>(Arc<Container>, PhantomData<T>), Scope enum + ScopeHandler trait + RefreshScope (wire to a future /actuator/refresh), thiserror ContainerError { NoSuchBean{suggestions}, NoUniqueBean{candidates}, CircularDependency{chain} } with strsim-free fuzzy suggestions (hand-rolled, dep policy). API: register::<T>(scope)/register_instance/register_factory/bind::<I: ?Sized + 'static, T>() (TypeId::of::<dyn Trait>() is a valid key, so trait-object bindings + primary/order + resolve_all::<dyn Trait>() work), resolve::<T>/resolve_named/contains/registered_types/reset_instance/register_scope. Idiom adaptations: constructor injection -> either hand-written factory closures (v1) or a firefly-container-macros sub-crate with #[derive(Component)] + #[component(binds(dyn UserRepo), name = "...")] generating the factory, and linkme distributed slices replacing package scanning; Autowired field injection -> drop (anti-idiom; constructor-only); isinstance assignability in resolve_all -> explicit bindings only; REQUEST scope -> tokio task_local (matches existing context-propagation pattern); SESSION-scoped serialized beans -> drop, no analog; generic injection Repository[User] -> free via monomorphized TypeIds. Deps: std/parking_lot + thiserror + tokio (all in workspace catalog). Cycle detection via thread/task-local resolution stack, mirroring container.py. Est: core crate ~700 src + ~500 tests; macros sub-crate ~600 src + ~300 tests.
