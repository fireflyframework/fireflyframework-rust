# Gap report: pyfly `domain`

Verdict: **partial** · est. 650 LOC

## What it is
pyfly.domain is a tiny (~520 LOC, zero-dependency) DDD building-block kit mirroring Java's org.fireflyframework.starter.domain: Entity (identity-based equality), ValueObject (immutable marker), the NON-event-sourced AggregateRoot that collects transient DomainEvents for post-commit publication, a composable in-memory Specification pattern, a collection-like async DomainRepository protocol, and domain exceptions layered on the kernel's RFC-7807 BusinessException. It is deliberately distinct from pyfly.eventsourcing (the replayed/versioned aggregate). Some of its surface is Python-class machinery with no Rust analog: ValueObject.replace() maps to native struct-update syntax, frozen-dataclass immutability is Rust's default, and __eq__/__hash__ overrides plus runtime_checkable Protocol checks are interpreter mechanics.

## pyfly public surface
Entity[TID] (id, is_transient, identity __eq__/__hash__); ValueObject (replace()); AggregateRoot[TID] (raise_event, pending_events -> copy, clear_events -> drain); DomainEvent (auto event_id uuid4, occurred_at UTC, event_type = class name); Specification[T] (is_satisfied_by, & | ~ combinators, __call__, Specification.of(callable, name=)); DomainRepository[A, TID] protocol (async add/find/remove/next_id); DomainException, BusinessRuleViolation(rule, code=DOMAIN_RULE_VIOLATION, context{rule}), AggregateNotFound(aggregate_type, id, code=DOMAIN_AGGREGATE_NOT_FOUND)

## Missing in Rust
Verified absent across all 52 crates (grep for Specification/is_satisfied/ValueObject/BusinessRule/pending_events returns nothing): (1) the composable in-memory Specification pattern (and/or/not combinators + closure adapter); (2) an Entity identity abstraction (id-based equality contract, is_transient); (3) the non-event-sourced aggregate primitive — a transient DomainEvent (auto uuid + timestamp, no version/wire format) plus a pending-events buffer with raise/snapshot/drain semantics; crates/eventsourcing's AggregateRoot is the *event-sourced* variant (version-stamped, base64 wire DomainEvent, EventStore-coupled) and corresponds to pyfly.eventsourcing, not pyfly.domain; (4) named domain error constructors: FireflyError has not_found/conflict/validation helpers but no business_rule(rule) -> DOMAIN_RULE_VIOLATION or aggregate_not_found(type, id) -> DOMAIN_AGGREGATE_NOT_FOUND with the structured {rule, aggregate_type, id} fields. Covered elsewhere: DomainRepository semantics map onto firefly-data's Repository<T,K> (save~add, find_by_id~find, delete~remove; next_id is Uuid::new_v4()); DomainException base semantics map onto firefly-kernel's FireflyError/ProblemDetail. Excluded as Python-runtime machinery: ValueObject.replace (struct-update syntax), frozen-dataclass marker, runtime Protocol checks.

## Porting plan
Extend crates/kernel with a new `ddd` module (pyfly.domain is zero-dep and 'importable from any layer' — exactly kernel's role), re-exported from firefly-starter-domain for discoverability; no new crate needed. Key items: (1) `Specification<T>` trait { fn is_satisfied_by(&self, candidate: &T) -> bool } with provided .and()/.or()/.not() combinators returning AndSpec/OrSpec/NotSpec, a blanket impl for Fn(&T) -> bool closures (replaces Specification.of), and optionally BitAnd/BitOr/Not on a Spec<T> newtype for operator parity (~130 LOC). (2) `Entity` trait { type Id: PartialEq + Hash; fn id(&self) -> Option<&Self::Id>; provided is_transient() and same_identity(&other) } — consumers implement PartialEq via the helper instead of a base-class __eq__ (~40 LOC). (3) `PendingEvents<E>` buffer struct (raise/pending -> &[E] / drain -> Vec<E>) generic over a per-aggregate event enum (idiomatic), plus an `EventMeta` struct (event_id: Uuid v4, occurred_at: DateTime<Utc>) and a `TransientDomainEvent` trait with event_type() defaulting via std::any::type_name, with a Box<dyn> fallback for heterogeneous buses (~120 LOC). (4) In kernel errors.rs add FireflyError::business_rule(rule, detail) (code DOMAIN_RULE_VIOLATION, 422, field rule) and FireflyError::aggregate_not_found(aggregate_type, id) (code DOMAIN_AGGREGATE_NOT_FOUND, 404, fields aggregate_type/id) (~40 LOC). (5) Re-exports + docs in firefly-starter-domain (~20 LOC). Deps: uuid + chrono only, both already in the workspace catalog. DomainRepository needs no port — document the mapping to firefly_data::Repository<T,K>. ~350 source LOC plus ~300 LOC of workspace-style inline tests porting tests/domain/* contracts (combinator composition, pending-events copy/drain, event auto-id, error code/field assertions).
