# Gap report: pyfly `cache`

Verdict: **partial** · est. 1400 LOC

## What it is
pyfly.cache is the framework's pluggable cache abstraction: an async CacheAdapter protocol, a primary+fallback CacheManager with automatic failover and write mirroring, three fully working adapters (InMemoryCache with TTL+LRU+stats, RedisCacheAdapter over redis.asyncio, PostgresCacheAdapter over async SQLAlchemy with lazy DDL) — there is no Hazelcast adapter in pyfly. It also ships Spring-style declarative decorators (@cacheable with condition/unless and null-caching, @cache_evict, @cache_put), a framework-type-tolerant JSON serializer (datetime/Decimal/UUID/set/Pydantic), DI auto-configuration with provider detection, and a put/get/evict round-trip health indicator. The Redis and Postgres adapters are full implementations, not scaffolds: both have unit tests against fakes plus docker-gated integration tests against real Redis/Postgres (tests/integration/test_cache_redis_integration.py, test_cache_postgres_integration.py). Python-runtime-specific machinery (decorator key templates resolved via inspect.signature, AutoConfiguration.is_available import-probing, async start/stop lifecycle hooks) has no meaningful Rust analog and is excluded from the gap.

## pyfly public surface
CacheAdapter protocol: get(key), put(key, value, ttl), put_if_absent(key, value, ttl)->bool, evict(key)->bool, evict_by_prefix(prefix)->int, exists(key)->bool, clear(), start(), stop(). CacheManager(primary, fallback) implementing the same protocol with failover/mirroring. Adapters: InMemoryCache(max_size) with get_stats()/get_keys(); RedisCacheAdapter(client) with get_stats()/get_keys(pattern, limit)/is_available(); PostgresCacheAdapter(engine) with get_stats()/get_keys(). Decorators: cache/cacheable(backend, key, ttl, condition, unless), cache_evict(backend, key, all_entries), cache_put(backend, key, ttl). serialization.cache_dumps/cache_loads. CacheAutoConfiguration (config keys pyfly.cache.enabled/provider/redis.url/postgres.url/max-size). CacheHealthIndicator.health() -> UP/OUT_OF_SERVICE/DOWN with latencyMs.

## Missing in Rust
crates/cache covers the port (Adapter), Memory, NoOp, Fallback (== CacheManager semantics), Typed<T>.get_or_set (== @cacheable idiom), and starter-core wires a cache health indicator. Genuinely missing: (1) RedisAdapter — pyfly ships a full working redis.asyncio adapter with integration tests; the Rust README/lib.rs explicitly defer it ('ships in the next minor') and no redis crate exists in the workspace catalog. (2) PostgresCacheAdapter — full SQL-table-backed cache (lazy DDL, ON CONFLICT upsert, TTL-aware reads, LIKE prefix eviction); no Rust counterpart and no Postgres driver in the catalog (only rusqlite). (3) Adapter trait operations: set_if_absent (NX semantics, used for locking/idempotency), exists, delete_prefix (relied on by pyfly transactional persistence and CQRS invalidation; Rust's CQRS QueryCache has prefix invalidation but only on its own private in-memory map, not via the Adapter port), stats (hits/misses/evictions/hit_rate/size). (4) MemoryAdapter lacks the LRU max_size bound and hit/miss counters that InMemoryCache(max_size) provides. (5) Config-driven provider selection (provider=memory|redis|postgres) in starter wiring. Not gaps: decorator sugar (Typed::get_or_set + explicit evict calls are the Rust idiom), Pydantic-tolerant serializer (serde covers it), null-caching workaround (Rust's NotFound-vs-value design already distinguishes a cached null), start/stop lifecycle (constructors/Drop + health_check).

## Porting plan
1) Extend crates/cache (firefly-cache): add to the Adapter trait, with default implementations to preserve object safety and existing impls — async fn set_if_absent(&self, key, value, ttl) -> Result<bool, CacheError> (default get-then-set, non-atomic; backends override), async fn exists(&self, key) -> Result<bool, CacheError> (default via get), async fn delete_prefix(&self, prefix) -> Result<u64, CacheError> (default Err(Backend("unsupported"))), fn stats() -> Option<CacheStats> with a new CacheStats { size, hits, misses, evictions, hit_rate } struct. Give MemoryAdapter an optional max_entries LRU bound (move-to-back on read over an ordered map or indexmap) and AtomicU64 hit/miss/eviction counters; override the new trait methods natively. Propagate the new ops through FallbackAdapter (mirror writes, union reads, sum prefix-evictions — exactly pyfly's CacheManager contract). ~300 LOC. 2) New crate crates/cache-redis (firefly-cache-redis), following the workspace's adapter-crate pattern (cf. idp-keycloak, ecm-storage-aws): RedisAdapter implementing Adapter using the `redis` crate with tokio-comp + connection-manager features (new workspace-catalog entry; nothing existing suffices). SET PX/NX for set/set_if_absent, SCAN MATCH loop for delete_prefix and a keys(pattern, limit) helper, EXISTS, DBSIZE for stats, FLUSHDB for clear, PING for health_check; constructor takes a connection URL or ConnectionManager (replaces pyfly's injected client + start/stop). ~300 src + 150 test LOC; docker-gated integration test mirroring tests/integration/test_cache_redis_integration.py (unit tests via a small in-crate fake implementing the redis ops or cfg-gated). 3) New crate crates/cache-postgres (firefly-cache-postgres): PostgresAdapter over tokio-postgres (or sqlx-postgres; either is a new catalog dep — the workspace currently has only rusqlite) porting the pyfly_cache_entries table, lazy DDL on first use, INSERT .. ON CONFLICT DO UPDATE / DO NOTHING, expires_at-aware SELECTs, LIKE-with-ESCAPE prefix deletion, COUNT(*) stats. ~350 src + 150 test LOC. 4) starter-core: extend FireflyConfig wiring so cache selection is config-driven (firefly.cache.provider = memory|redis|postgres) behind cargo features `redis`/`postgres` — the Rust adaptation of CacheAutoConfiguration's import-probing. ~120 LOC. Decorators map to existing Typed::get_or_set plus explicit adapter.delete/delete_prefix calls after mutations; optionally add Typed::get_or_set_unless(predicate) (~30 LOC) for @cacheable(unless=...) parity.
