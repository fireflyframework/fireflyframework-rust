# Gap report: pyfly `actuator+observability+resilience+scheduling+security`

Verdict: **partial** · est. 5800 LOC

## What it is
Five pyfly packages providing Spring Boot operational parity: (1) actuator — 15 management endpoints with Spring's web-exposure model and an extensible endpoint registry; (2) observability — labeled Prometheus metrics, @timed/@counted/@span decorators, OTLP tracer auto-config, W3C trace-context propagation, correlation/request/tenant contextvars, process/system gauges; (3) resilience — circuit breaker (consecutive + windowed failure-rate modes), retry, token-bucket rate limiter, bulkhead, time limiter, fallback, and a config-driven named registry; (4) scheduling — @scheduled with 5/6-field cron (seconds, '?', IANA zones), fixed-rate/fixed-delay/initial-delay, and ShedLock-style distributed locks (Redis SET-NX-PX, Postgres advisory); (5) security — JWT/bcrypt, SpEL-subset method security, role hierarchy, HttpSecurity URL DSL, CSRF, and a full OAuth2 stack (JWKS resource server, auth-code+PKCE login, client_credentials/refresh authorization server, pluggable token stores). All are FULL implementations backed by ~60 test modules. Python-runtime-specific machinery with no Rust analog: decorator metadata scanning, sync/async dual wrappers, contextvars plumbing, prometheus_client global-registry idempotency, threaddump of Python frames, and the DI-container-introspection endpoints (beans/conditions/configprops/mappings) which presuppose pyfly's bean container.

## pyfly public surface
actuator: ActuatorEndpoint, ActuatorRegistry, HealthAggregator/HealthIndicator/HealthResult/HealthStatus/ProbeGroup, build_actuator_routes, install_health_indicators + endpoints {health(+/liveness,/readiness,/{group|component}), info, metrics(+/{name}?tag=k:v), prometheus, loggers(GET/POST), env(+/{prop}), beans, configprops, mappings, scheduledtasks, conditions, caches(+/{name}), refresh, threaddump, httpexchanges} + exposure(include/exclude/base-path). observability: MetricsRegistry/MetricsRecorder/NoOpMetricsRecorder, timed, counted, span, bind/unbind_correlation_context + get/set {correlation,request,tenant}_id/traceparent/tracestate, extract_context/inject_headers, ProcessMetricsCollector. resilience: CircuitBreaker/CircuitState, retry, RateLimiter, Bulkhead, time_limiter, fallback, ResilienceRegistry.from_config. scheduling: scheduled(cron|fixed_rate|fixed_delay, initial_delay, zone, lock, lock_ttl), CronExpression(next/prev/next_n/seconds_until), TaskScheduler, DistributedLock/LocalLock/InProcessDistributedLock + Redis/Postgres lock adapters, TaskExecutorPort. security: SecurityContext, JWTService, PasswordEncoder/BcryptPasswordEncoder, secure/pre_authorize/post_authorize, evaluate_security_expression, RoleHierarchy, HttpSecurity DSL (AccessRule/SecurityRule), CSRF helpers, JWKSTokenValidator, ClientRegistration(+google/github/keycloak), OAuth2LoginHandler, AuthorizationServer, TokenStore (InMemory/Redis/Postgres), OAuth2SessionSecurityFilter.

## Missing in Rust
actuator (crates/actuator has 6 endpoints vs 15): no liveness/readiness probe groups or named health groups, no per-component health drill-down or show-details/show-components config, no loggers endpoint (runtime log-level read/write), no scheduledtasks, caches, refresh, or httpexchanges endpoints, no Spring exposure model (include/exclude/base-path/per-endpoint enabled), no extensible endpoint trait/registry, metrics endpoint is Prometheus-text-only with unlabeled counters/gauges (no histograms/timers, no labels, no Micrometer JSON view with tag drill-down). observability: no labeled metrics or histogram/timer primitives, no timed/counted instrumentation wrappers, no OpenTelemetry/OTLP wiring (explicitly left to apps), no W3C traceparent/tracestate extract+inject propagation, kernel task-local covers correlation id only (no tenant-id/request-id), no process/system gauges (process_uptime_seconds etc.). resilience (mostly covered: breaker/rate-limiter/bulkhead/timeout/Chain + utils::retry): circuit breaker lacks the count-based failure-rate-window mode and configurable half-open probe budget; no config-driven named ResilienceRegistry; no fallback decorator (minor — Result combinators suffice). scheduling: cron is 5-field only (no Spring 6-field seconds, no '?', no macros), no IANA timezone per task (Local time hardwired), no initial_delay, and no distributed-lock support at all (no ShedLock port, no Redis/Postgres adapters). security (largest gap): no JWKS-based OAuth2 resource-server verifier (idp-keycloak/azure-ad/aws-cognito are declared not-implemented stubs; only internal-db HMAC JWT works), no OAuth2 login flow (auth-code/state/nonce/PKCE), no OAuth2 authorization server or TokenStore adapters, no client-registration presets, no RoleHierarchy, no CSRF double-submit helpers, FilterChain matches path prefixes only (no glob patterns, no deny_all, no permission/authority rules), no pre/post-authorization guard capability (pyfly's string SpEL is a Python idiom, but the capability itself — arg- and return-value-aware authorization checks — is absent).

## Porting plan
Extend existing crates; no new crate needed. (1) crates/actuator (~1,400 LOC): add ProbeGroup to HealthIndicator registration + /actuator/health/liveness|readiness|{group} routes and component drill-down; ActuatorConfig gains exposure: {include, exclude, base_path} honored in mount(); add an `Endpoint` trait (id() + handle(selector, query) -> Result<Value>) + registry so apps can mount custom endpoints; add LoggersEndpoint backed by tracing_subscriber::reload::Handle<EnvFilter> (GET levels via stored directives, POST set/reset); add ScheduledTasksEndpoint reading a task-descriptor list exposed by firefly-scheduling::Scheduler; add HttpExchanges tower layer (bounded VecDeque<Exchange>, header masking) + endpoint; upgrade MetricRegistry with label support and Histogram (fixed buckets) so /actuator/metrics can serve the Micrometer JSON shape with availableTags and keep /actuator/prometheus text. Skip beans/conditions/configprops/mappings (DI-container introspection — no DI container in the Rust port by design) and threaddump (covered by /actuator/tasks). (2) crates/observability (~1,100 LOC): labeled Counter/Gauge/Histogram + timer helper (record closure/future duration with class/method/exception-style labels — explicit `timed("name", fut)` fns instead of decorators); W3C propagation module (parse/format traceparent+tracestate, tower extract layer + reqwest inject helper); extend firefly-kernel task-locals with tenant_id/request_id mirroring correlation_id; ProcessMetricsCollector (uptime/start-time/cpu via /proc + available_parallelism); optional `otel` feature wiring tracing-opentelemetry + OTLP exporter from config keys. (3) crates/resilience (~450 LOC): CircuitConfig gains failure_rate_threshold + window_size (count-based ring) + half_open_max_calls; new registry.rs with ResilienceRegistry::from_config(&firefly_config::Config) materializing named breakers/limiters/bulkheads/timeouts (reuse humantime for durations); optional Fallback decorator for Chain. (4) crates/scheduling (~800 LOC): extend parse_cron to optional 6-field seconds-first + '?' + @hourly-style macros; CronTrigger<Tz: TimeZone> with chrono-tz IANA zone (default Utc); initial_delay on fixed-rate/fixed-delay builders; new lock.rs `#[async_trait] trait DistributedLock { try_acquire(name, ttl) -> bool; release(name) }` + InProcessLock, RedisLock (SET NX PX + owner-token Lua release, feature `redis`), PostgresAdvisoryLock (pg_try_advisory_lock over sqlx, feature `postgres`); Task gains optional lock name/ttl checked per tick. (5) crates/security (~2,000 LOC): JwksVerifier implementing the existing Verifier trait (reqwest JWKS fetch + kid cache + jsonwebtoken RS256, issuer/audience validation, Keycloak realm_access.roles + scope->authorities mapping — also unblocks the idp stubs); RoleHierarchy (parse "A > B" spec, transitive expansion, consulted by FilterChain); FilterChain upgrades: glob patterns (globset), deny(), authenticated(), require_authority(); add `authorities: Vec<String>` to Authentication; csrf.rs (token gen + constant-time validate + double-submit tower layer); oauth2 module: ClientRegistration + google/github/keycloak presets, axum login router (authorization-code + state/nonce + PKCE S256, tower-sessions storage), AuthorizationServer (client_credentials + refresh_token grants, jsonwebtoken HS256) with `#[async_trait] TokenStore` + InMemory/Redis/Postgres impls. Replace pyfly's string SpEL with typed guard fns (`require(expr: impl Fn(&Authentication) -> bool)`) rather than porting the AST evaluator. Workspace catalog already has jsonwebtoken, bcrypt, reqwest, redis, sqlx, tower, axum, chrono — add chrono-tz and globset.
