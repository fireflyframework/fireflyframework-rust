# Gap report: pyfly `pyfly core`

Verdict: **partial** · est. 800 LOC

## What it is
pyfly "core" is the application-bootstrap package of the Python port: PyFlyApplication runs a Spring-Boot-parity startup sequence (config load with framework/starter defaults + profile overlays, optional remote Spring-Cloud-Config import, logging setup, DI context start, banner, structured startup/route/server logs, graceful shutdown). It also owns the framework's hierarchical Config (dot-notation get, PYFLY_* env overrides with type coercion, ${...} placeholder resolution, relaxed kebab/snake binding to dataclasses/Pydantic via @config_properties, runtime reload, Spring-Sanitizer-style property-source masking), the BannerPrinter (TEXT/MINIMAL/OFF + custom banner file), a safe SpEL-lite #{...} expression evaluator, and the @Value injection descriptor. Some of it is Python-DI machinery (class-decorator app marking, package scanning, bean/wiring summaries, descriptor-based field injection, uvicorn/granian worker env-var plumbing) with no meaningful Rust analog.

## pyfly public surface
PyFlyApplication (startup()/run()/shutdown(), .context, .startup_time_seconds, remote-config import, starter-defaults collection), @pyfly_application(name, version, scan_packages, description) decorator, BannerPrinter + BannerMode (TEXT/MINIMAL/OFF, custom location, ${...} banner placeholders), Config (from_sources/from_file, get with PYFLY_* env override + type coercion, get_section/effective_section/effective_dict, bind to @config_properties dataclasses/Pydantic with relaxed kebab↔snake binding, reload_from_sources, property_sources with origin + mask_value Sanitizer, ${key:default} placeholder resolution), @config_properties(prefix) decorator, Value descriptor (${key}, ${key:default}, #{expr}), expression.evaluate/is_expression/ExpressionError (SpEL-lite).

## Missing in Rust
Bootstrap/runner/banner are covered (crates/lifecycle Application + crates/starter-core Core + crates/observability banner). The gaps are all in the configuration subsystem (crates/config + crates/config-server): (1) no ${key}/${key:default}/${ENV_VAR} placeholder resolution in config values anywhere in the workspace (pyfly resolves recursively with a depth-10 circular-reference guard, env beats config refs); (2) no runtime config reload — pyfly's Config.reload_from_sources() replays the exact source merge and atomically swaps (backs the /actuator/refresh + refresh-scoped beans contract); (3) no property-source introspection with sensitive-value masking — pyfly's property_sources() returns ordered sources (systemEnvironment first) with per-key value+origin, masking password/secret/token/key/credential keys and redacting passwords inside URI userinfo (Spring Sanitizer parity); the Rust actuator /actuator/env only redacts env vars outside an allow-prefix list; (4) no relaxed (kebab-case ↔ snake_case) key binding — Rust only lower-cases keys, so `graceful-timeout:` in YAML cannot bind a `graceful_timeout` serde field; (5) single active profile via FIREFLY_PROFILE vs pyfly's comma-separated multi-profile list resolvable from the config file itself (pyfly.profiles.active) as well as env; (6) no TOML config source (pyfly supports pyfly.toml alongside pyfly.yaml); (7) no Spring-Cloud-Config CLIENT — crates/config-server serves the wire format but nothing fetches remote config and merges it at boot like pyfly's _import_remote_config (uri/label/username/password, fail-fast option, non-fatal fallback); (8) minor: banner has no mode (TEXT/MINIMAL/OFF) or custom-banner-file location config. Excluded as Python-runtime/DI-specific (no Rust analog by design): @pyfly_application class decorator + package scanning, bean/stereotype/wiring-summary startup logs, the Value descriptor (@Value field injection), the SpEL-lite #{...} evaluator (its only consumers are @Value and @ConditionalOnExpression DI conditions; Rust expresses conditional wiring in plain code), and the _PYFLY_* uvicorn/granian worker-process env plumbing.

## Porting plan
Extend crates/config (firefly-config) — no new crate needed: (a) placeholder.rs: post-merge pass over the flat map resolving ${key}, ${key:default}, ${ENV_VAR} with env-beats-config precedence and a recursion depth guard; call from load()/bind() (~180 LOC w/ tests, no new deps — regex is in the workspace catalog); (b) relaxed keys: normalize '-'→'_' in Layered::merge alongside the existing lowercase so kebab YAML binds snake serde fields (~50 LOC); (c) reload.rs: ReloadableConfig<T: DeserializeOwned> holding Vec<Box<dyn Source>> + RwLock<Arc<T>> with reload() replaying merge+bind and atomically swapping (std sync only, ~140 LOC), exposing a hook firefly-actuator can wire as /actuator/refresh; (d) introspection: Layered::property_sources() -> Vec<PropertySourceView{name, properties: {key -> {value, origin}}}> (reverse merge order, env source first) plus a mask module porting the sensitive-key heuristics and URI-userinfo regex redaction; feed into firefly-actuator's env handler as an alternative to the allow-prefix redaction (~180 LOC); (e) active_profiles() -> Vec<String> from comma-separated FIREFLY_PROFILE + multi-overlay in profile_sources (~30 LOC); (f) optional TomlSource mirroring YamlSource (needs `toml` added to the workspace dep catalog — defer if TOML parity is not required, Java/Go ports don't have it). Add a client to crates/config-server behind a `client` feature (reqwest from the catalog): ConfigClient{url, application, profile, label, basic-auth} with async fetch() flattening propertySources to HashMap<String,String>, plus a StaticSource bridge so the fetched map slots into the Layered chain with a fail-fast/soft-fallback option mirroring pyfly (~180 LOC w/ tests). Optionally extend firefly-observability render_banner with BannerMode::{Text,Minimal,Off} + custom template path (~70 LOC). Idiom adaptations: @config_properties decorator -> #[derive(Deserialize)] + firefly_config::load (already the pattern); @Value/#{...} -> read bound struct fields / plain Rust conditionals (do not port); reload-scoped beans -> ReloadableConfig handle cloned into handlers.
