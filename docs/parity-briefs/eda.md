# Gap report: pyfly `eda`

Verdict: **partial** · est. 2000 LOC

## What it is
pyfly.eda is the event-driven-architecture module: a wire-compatible EventEnvelope, an EventPublisher port (subscribe by fnmatch event-type pattern, publish to a destination), and five adapters — InMemoryEventBus plus FULLY WORKING KafkaEventBus (aiokafka), RabbitMqEventBus (aio-pika, durable exchange/queues, manual ack/nack at-least-once), PostgresEventBus (asyncpg outbox table + LISTEN/NOTIFY, per-group offsets, advisory-lock single drainer), and RedisStreamsEventBus (XREADGROUP consumer groups). Support layer: JSON serializer (Avro/Protobuf are NotImplementedError stubs even in pyfly), event filters, in-memory DLQ store, event circuit breaker, actuator health indicator, @event_listener/@event_publisher/@publish_result decorators, and config-keyed provider auto-configuration with installed-library auto-detection. The decorator/bean auto-discovery machinery is Python DI plumbing (maps to explicit subscribe calls in Rust) and the auto-detection of installed libraries has no Cargo analog.

## pyfly public surface
EventEnvelope, ErrorStrategy enum; EventPublisher protocol (subscribe(pattern, handler), publish(destination, event_type, payload, headers), start, stop) + EventHandler; adapters: InMemoryEventBus, KafkaEventBus, RabbitMqEventBus, PostgresEventBus, RedisStreamsEventBus; EventSerializer/JsonEventSerializer (+ Avro/Protobuf stubs); EventFilter/HeaderEventFilter/PredicateEventFilter; EdaDeadLetterEntry/EdaDeadLetterStore/InMemoryEdaDeadLetterStore; EventCircuitBreaker/CircuitBreakerConfig/CircuitOpenError; event_listener/event_publisher/publish_result decorators; EventPublisherHealthIndicator; EdaAutoConfiguration (provider = memory|kafka|redis|postgres|rabbitmq|auto, serialization-format selection)

## Missing in Rust
The key asymmetry: pyfly ships WORKING Kafka, RabbitMQ, Postgres, and Redis Streams adapters (real aiokafka/aio-pika/asyncpg/redis client code with consumer groups, ack/nack, outbox+offsets), while Rust crates/eda has only the in-memory broker full; new_kafka_broker/new_rabbitmq_broker are deliberate sentinel-error scaffolds and there is no Postgres or Redis transport even as a scaffold (no KafkaConfig-style struct, no factory). Also missing: (1) wildcard/pattern subscription — pyfly dispatches on fnmatch event_type patterns, Rust Subscriber is exact-topic only; (2) generic EDA dead-letter store abstraction (webhooks crate has a private DLQ, eda has none); (3) broker health indicator wiring (actuator HealthIndicator trait exists but no EDA indicator); (4) config-driven provider selection — starter-core CoreConfig takes an injected Arc<dyn Broker> defaulting to InMemoryBroker but never reads an eda provider key from firefly-config. NOT gaps: Avro/Protobuf serializers (stubs in pyfly too), circuit breaker (firefly-resilience covers it), JSON envelope encoding (Event serde already wire-compatible), decorators/bean auto-discovery (Python DI plumbing; Rust idiom is explicit subscribe). The workspace dependency catalog contains no Kafka/AMQP/async-Postgres/Redis client crates, so all four transports need new third-party deps.

## Porting plan
Follow the crate names the eda README already reserves: new crates firefly-eda-kafka, firefly-eda-rabbitmq, plus firefly-eda-postgres and firefly-eda-redis, each exposing a constructor returning Box<dyn Broker> (impl Publisher + Subscriber from firefly-eda); keep crates/eda dependency-free and have starters call the transport crate's factory in place of the sentinel. (a) firefly-eda-kafka: KafkaBroker over rdkafka FutureProducer + StreamConsumer (workspace dep rdkafka ~0.36; note librdkafka C dep — pure-Rust rskafka is the alternative), consumer-group loop deserializing Event JSON and dispatching to handlers, ~350 LOC. (b) firefly-eda-rabbitmq: RabbitMqBroker over lapin — declare durable direct exchange, queue '<group>.<destination>' bound per destination, basic_consume with manual ack / nack(requeue=true) on handler error / reject(requeue=false) on undeserializable body, ~400 LOC. (c) firefly-eda-postgres: PostgresBroker over sqlx PgPool + PgListener — port the DDL (firefly_eda_outbox BIGSERIAL outbox, firefly_eda_offsets cursor table), publish = INSERT + NOTIFY, drain loop gated by pg_try_advisory_lock on a SHA-256-folded i64 group key, advance cursor only after successful dispatch (at-least-once), poll-interval fallback wake, ~600 LOC. (d) firefly-eda-redis: RedisStreamsBroker over redis crate (tokio-comp + streams) — XGROUP CREATE MKSTREAM, XREADGROUP block loop, XACK on success, leave pending on handler error, ~350 LOC. (e) extend crates/eda: glob pattern matching on event_type for handler dispatch (small fnmatch impl or globset; add subscribe_pattern or match inside transports as pyfly does), DeadLetterStore trait + DeadLetterEntry + InMemoryDeadLetterStore, ErrorStrategy enum, PostgresConfig/RedisConfig structs mirroring KafkaConfig, ~200 LOC. (f) extend starter-core: EdaSettings read from firefly-config (provider key, destinations, group) selecting the broker factory — explicit feature-gated wiring replaces pyfly's import-probing auto-detection — plus an actuator IndicatorFn over Arc<dyn Broker>, ~100 LOC. Decorators map to explicit broker.subscribe / explicit publish calls; no porting needed. Circuit breaker: reuse firefly-resilience.
