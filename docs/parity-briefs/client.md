# Gap report: pyfly `client`

Verdict: **partial** · est. 700 LOC

## What it is
pyfly.client is the outbound service-client subsystem: a declarative Spring-style REST client (@service_client/@http_client class decorators + @get/@post/@put/@delete/@patch method decorators, wired at container startup by HttpClientBeanPostProcessor over an httpx adapter) with built-in resilience (CircuitBreaker outside, RetryPolicy with exponential backoff inside, retrying only transient 429/5xx/connection/timeout failures) and an HTTP-status-to-typed-exception mapper. It also ships four FULL (thin but real, live-transport-tested) protocol clients: GraphQLClient (POSTs query/variables/operationName, raises on errors, returns data), SoapClient (SOAP 1.1 envelope builder + POST with SOAPAction, no WSDL), GrpcClientBuilder (secure/insecure grpc.aio channel with options), and WebSocketClient (connect + send/stream async iterator with ping config). Python-runtime machinery with no Rust analog: the decorator/BeanPostProcessor metaprogramming, @auto_configuration conditional bean registration, and PooledHttpClient (an async-context-manager shim; reqwest::Client is already pooled/clone-shared).

## pyfly public surface
CircuitBreaker, CircuitState; RetryPolicy; service_client, http_client, get, post, put, delete, patch (decorators); HttpClientBeanPostProcessor; HttpClientPort (port) + HttpxClientAdapter; ServiceClientException hierarchy (Validation/Authentication/NotFound/Conflict/UnprocessableEntity/RateLimit/Unavailable) + map_http_error; GraphQLClient + GraphQLClientBuilder; SoapClient + SoapClientBuilder; GrpcClientBuilder (channel()); WebSocketClient + WebSocketClientBuilder (connect()/stream()); PooledHttpClient; ClientAutoConfiguration/GrpcClientAutoConfiguration/WebSocketClientAutoConfiguration.

## Missing in Rust
REST is fully covered: crates/client RestBuilder/RestClient gives JSON encode/decode, default headers, timeout, exponential-backoff retry on exactly the same transient set (network errors + 429/5xx), correlation-id propagation, and RFC 7807 decode into FireflyError (status code available, so pyfly's per-status exception classes map to matching on ferr.status — no gap). Circuit breaker exists in firefly-resilience (CircuitBreaker/CircuitState) and composes with the client. Genuinely missing: (1) GraphQL client — no counterpart anywhere in the workspace (pyfly's is FULL, respx-transport-tested); (2) SOAP client — Rust new_soap() is a placeholder returning ClientError::TransportNotRegistered, pyfly's is FULL (envelope build + POST + SOAPAction); (3) WebSocket client — Rust new_websocket() is the same placeholder, pyfly's is FULL (connect with headers/ping, stream iterator, live echo-server tested); (4) gRPC channel builder — Rust new_grpc() is the placeholder, pyfly's is a FULL thin secure/insecure channel factory with options (live in-process server tested). Also minor: pyfly's httpx adapter injects W3C traceparent on every outbound request; Rust RestClient forwards only X-Correlation-Id. NOT gaps (Python machinery): declarative @get/@post interface generation, BeanPostProcessor wiring, auto-configuration conditionals, PooledHttpClient.

## Porting plan
Extend crates/client (firefly-client) rather than new crates — pyfly's protocol clients are deliberately thin, so feature-gated modules fit (scaffold.rs already names the planned homes; keep TransportNotRegistered when a feature is off). (a) graphql.rs (no new deps, reqwest+serde_json): GraphQlBuilder { endpoint, headers, timeout } -> GraphQlClient::execute<V: Serialize, T: DeserializeOwned>(query, variables, operation_name) -> Result<T, ClientError>; add ClientError::GraphQl(Vec<serde_json::Value>) for the errors array; ~120 LOC. (b) soap.rs (no new deps): SoapBuilder { endpoint, soap_action, headers, timeout } -> SoapClient::call(body_xml: &str) -> Result<String, ClientError>, wrapping the SOAP 1.1 envelope and POSTing text/xml with SOAPAction header — mirror pyfly's no-WSDL 80% case, no quick-xml needed; ~110 LOC. (c) websocket.rs (add tokio-tungstenite to the workspace catalog, feature \"websocket\"): WsBuilder { url, headers, ping_interval } -> WsClient::connect() -> WebSocketStream plus a stream(send: Vec<Message>) -> impl Stream<Item = Result<Message, _>> helper; ~170 LOC. (d) grpc.rs (add tonic + tonic feature \"tls\", feature \"grpc\"): GrpcBuilder { target, secured, options } -> connect() -> tonic::transport::Channel for caller-supplied generated stubs — channel-only, like pyfly; ~110 LOC. (e) Replace scaffold placeholders/re-exports and optionally inject traceparent alongside X-Correlation-Id in RestClient (~80 LOC). Idiom adaptations: declarative decorators -> explicit RestClient::request calls; @service_client auto-resilience -> compose via firefly_resilience::{CircuitBreaker, Chain} (optionally add a RestBuilder::with_circuit_breaker convenience); typed per-status exceptions -> match on FireflyError.status; auto-configuration -> direct builder construction (or starter-core registration). Integration tests mirroring pyfly's live tests (axum WS echo, tonic in-process echo, mock GraphQL/SOAP endpoints) add ~400 LOC on top of the ~590 source LOC.
