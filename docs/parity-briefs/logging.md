# Gap report: pyfly `logging`

Verdict: **partial** · est. 880 LOC

## What it is
pyfly.logging is the hexagonal logging subsystem: a LoggingPort (configure/get_logger/set_level) with a default StructlogAdapter and a zero-dependency StdlibLoggingAdapter, fully driven by LoggingProperties (pyfly.logging.*): per-logger level map, format console|json|logfmt, logback-style layout patterns, size-rotating file appender, and an external config file. It enriches every record with trace_id/span_id from the active OTel span (the MDC equivalent) and ships a PII redaction subsystem: a regex engine with 10 built-in entities (EMAIL, CREDIT_CARD with Luhn validation, IBAN, US_SSN, JWT, BEARER_TOKEN, URL_CREDENTIALS, PHONE, IPV4, IPV6), three mask styles (placeholder/partial/hash), extra patterns, allow/deny field lists, an optional Presidio NER engine (pyfly[pii] extra, always falls back to regex), and opt-in stdout/stderr stream redaction. Substantial machinery is Python-runtime-specific: structlog<->stdlib bridging (ProcessorFormatter, foreign_pre_chain), logging.config dictConfig/fileConfig loading, the get_logger structlog shim, logback-token-to-%-style pattern compilation, and sys.stdout/sys.stderr monkeypatch interception.

## pyfly public surface
LoggingPort (configure(config), get_logger(name), set_level(name, level)); StructlogAdapter; StdlibLoggingAdapter; get_logger(name); LoggingProperties (level: dict incl. root + named loggers, format: console|json|logfmt, pattern.console/file, file.name/path, rolling.max_size/max_history/total_size_cap, config path, redaction.*); redaction: Redactor protocol, RegexRedactor, PresidioRedactor, build_redactor, BUILTIN_PATTERNS + luhn_valid, make_structlog_redactor, RedactionFilter, RedactingTextIO/install_stream_redaction; layout.compile_pattern; handlers.parse_size/build_file_handler; config_loader.apply_external_config; structlog processor _add_trace_ids (trace_id/span_id from active span).

## Missing in Rust
crates/observability/src/logging.rs covers the core: CorrelationLayer with JSON and text(logfmt) renderers, correlationId enrichment from the firefly_kernel task-local, service attribute, span-field merging, pluggable writer, init_logging, and starter-core wires LogConfig with the app name. crates/web/src/pii.rs has mask_pii/mask_map but only as manual helpers (4 entities: email/iban/card/phone; single fixed placeholder style; no Luhn check, no JWT/bearer/URL-credential/SSN/IP patterns; not attached to the log pipeline). Genuinely missing for a migrating consumer: (1) per-logger/per-target level configuration — LogConfig has one global level; the pyfly level map ({root: INFO, "my.module": DEBUG}) has no equivalent (tracing-subscriber env-filter feature is in the workspace catalog but unused); (2) runtime level changes (LoggingPort.set_level) — no reload handle, and no /actuator/loggers endpoint (actuator exposes health/info/metrics/env/tasks/version only); (3) file appender with size-based rotation (max_size/max_history) — output is a single Write sink, stdout by default; (4) integrated, configurable PII redaction in the log pipeline: entity selection, extra_patterns, mask styles placeholder|partial|hash, Luhn validation, allow_fields/deny_fields, deny-field wholesale replacement; (5) trace_id/span_id log enrichment — only correlationId is injected; OTel wiring is explicitly left to the app; (6) config binding of logging settings — crates/config has no logging properties section, so logging cannot be driven from firefly.logging.* config like every other port. NOT counted as missing (Python-specific): dictConfig/fileConfig external-config loading, structlog/stdlib bridge, logback-pattern-to-%(...)s translation, sys.stdout/stderr interception, Presidio NER (Python-only library; regex is the cross-port contract and pyfly itself falls back to it).

## Porting plan
Extend crates/observability (no new crate). (a) Per-target levels + runtime set_level: add levels: BTreeMap<String, Level> to LogConfig consulted in CorrelationLayer::enabled/on_event by longest-prefix target match, plus a LevelHandle (Arc<RwLock<...>> shared with the layer, or tracing_subscriber::reload + EnvFilter — env-filter feature already in the workspace catalog) exposing set_level(target, level); ~150 LOC. (b) redaction.rs module: trait Redactor { fn redact(&self, &str) -> Cow<'_, str> }, RegexRedactor porting the 10 BUILTIN_PATTERNS + luhn_valid validator, enum MaskStyle { Placeholder, Partial, Hash } (sha2 in catalog), RedactionConfig { enabled, entities, mask, extra_patterns, allow_fields, deny_fields }; apply in CorrelationLayer::write_record to msg and string field values, replacing deny-listed keys with "<REDACTED>"; consolidate with/supersede firefly_web::mask_pii patterns; deps regex + sha2 only; ~300 LOC. (c) appender.rs: RollingFileWriter (impl Write) with parse_size("10MB") and max_history backup pruning via std::fs, plus a TeeWriter so console+file both receive records, and LogConfig.file: Option<FileConfig>; ~180 LOC. (d) trace-id enrichment: a pluggable Enricher hook on CorrelationLayer (fn(&mut FieldSet)) defaulting to the kernel correlation id, letting apps that wire tracing-opentelemetry inject trace_id/span_id — mirrors pyfly _add_trace_ids without forcing an OTel SDK dep; ~50 LOC. (e) LoggingProperties serde struct in crates/config (level map, format, file, rolling, redaction) + From<LoggingProperties> for LogConfig consumed by starter-core; ~120 LOC. (f) optional parity follow-up: GET/POST /actuator/loggers in crates/actuator backed by the LevelHandle; ~80 LOC. Idiom adaptations: LoggingPort protocol -> tracing facade + LogConfig builder (no port trait needed); structlog processors -> field processing inside the Layer; stdout monkeypatching -> redaction at the layer's writer boundary (document that raw println! from third-party crates is not interceptable in Rust); Presidio engine config value -> accept only "regex", warn otherwise.
